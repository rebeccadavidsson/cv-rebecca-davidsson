Logpoints: Debugging without console.log

  1. Open Sources panel
  2. Right-click on a line number ‚Üí Add logpoint
  3. Type a message and include variables {myVar}
  4. Reload: It's like a temporary console.log (no code changes required!)
// Open the 'Sources' tab and right-click here!
function add(a, b) {
  return a + b; // ‚Üê Set a logpoint here: "Adding: {a} + {b}"
}
add(10, 5);
Pro tip: Logpoints persist on page reloads and don't pollute production code.

Conditional Breakpoints

  1. In the Sources panel, right-click a line ‚Üí Add conditional breakpoint
  2. Enter a condition (e.g. a > 10)
  3. The script only pauses when that condition is met
for(let i=0;i<10;i++){
  // Set conditional breakpoint: i === 7
  console.log(i);
}

DOM Breakpoints: Monitor Live DOM Changes

  1. Right-click any element in Elements panel ‚Üí Break on...
  2. Choose:
    • Attribute modification
    • Subtree modification
    • Node removal
  3. On change, DevTools will pause right at the triggering script line

The debugger Statement

Drop debugger into your code. When Chrome hits it, it halts as if by breakpoint:


function demoPause() {
    debugger;
    alert("Paused in DevTools!");
}
